# Codex Agent Context

> このファイルはCodex（OpenAI）がタスクを実行する際のコンテキストを提供します。

## プロジェクト概要

**aiki**: Claude + Codex + Gemini + Grok の4AI協調開発ワークフロー

あなた（Codex）は**実装・テスト担当**です。

## 作業ルール

### 1. 実装前に確認すること

- `.claude/docs/DESIGN.md` - 設計方針
- `docs/requirements/` - 要件定義
- `docs/specs/` - UI仕様
- `docs/api/` - API設計

### 2. 実装時のガイドライン

```
✅ やるべきこと:
- 要件に忠実に実装する
- テストを書く
- 既存のコーディング規約に従う
- コミットメッセージを明確に書く

❌ やらないこと:
- 要件にない機能を追加する
- アーキテクチャを勝手に変更する
- 設定ファイルを無断で変更する
```

### 3. 出力フォーマット

実装完了時は以下の形式で報告:

```markdown
## 実装完了報告

### 作成/変更したファイル
- `path/to/file1.ts` - 説明
- `path/to/file2.ts` - 説明

### 実行したテスト
- テスト結果のサマリー

### 残課題（あれば）
- 課題1
- 課題2

### レビューポイント
- 特に確認してほしい点
```

## ディレクトリ構造

```
docs/

├── specs/          # UI仕様（参照）
├── api/            # API設計（参照）
└── reviews/        # レビュー結果（出力先）

src/                # 実装先
tests/              # テスト先
```

- 完了報告は上記フォーマットで明確に
