# Grok委譲ルール

## 適用条件

以下のタスクはGrokへの委譲を検討する：

1. **リアルタイムトレンド調査**: X/SNSの最新トレンドや議論
2. **最新技術アップデート**: 破壊的変更、新リリース、非推奨化
3. **コミュニティ感情分析**: 開発者の意見や反応
4. **コンテンツリサーチ**: 記事執筆のための一次情報収集
5. **バイラルコンテンツ分析**: 何がトレンドで、なぜか

## 委譲手順

1. 調査目的と期待するアウトプットを明確化
2. `.grok/GROK.md` のテンプレートを使用
3. 結果を `.claude/docs/research/` に保存するよう依頼
4. Claudeが結果を読み込んで意思決定に活用

## 委譲しないケース

- 大規模な静的コード分析（Geminiを使用）
- 実装タスク（Codexを使用）
- リアルタイム要素のない歴史的・アーカイブ調査
- セキュリティ上機密性の高い情報を含むケース

## コスト考慮

- Grok: xAI API従量課金
- リアルタイム・時間に敏感なクエリに最適
- Gemini（静的分析）をライブデータで補完
